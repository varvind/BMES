BMES Participation Tracker Help Page

This is the user documentation for the TAMU BMES Participation Tracker Website. TAMU BMES is the Biomedical Engineering Society at Texas A&M University. They have engaged our company in order to create a web application that they will use to track member attendance at organization events.

--------------------- Event Sign In Area---------------------

The website consists of two main areas, the event sign in area and the club officer area. Upon first visiting the website, the user should be taken to the home page, which appears as so:

Home Page:

Item 1A: The club officer login button. Left clicking on this button will redirect the user to the sign in page for the club officer area where they will be asked for both a username and password. If a club member/ individual without login credentials mistakenly navigates here, they will be able to return to the homepage with the back button in their web browser.

Item 1B: The events panel. This panel contains all of the events hosted by the club, sorted by date, with most recent dates displayed at the top of the list. Events that occurred over 2 days ago will disappear from this panel in order to help prevent the home page from becoming too cluttered, but all events will always be visible from the club officer area. Please note that each individual event has its own unique “Details” button and its own unique “Event Check-In” button.

Item 1C: The event details button. Left clicking on this button will redirect the user to the event details page, where they will be able to get a better view of the event information. Users will be able to return to the home page using the back arrow in their browser or by clicking the home button on the top left of the page.

Item 1D: The event check-in button. Left clicking on this button will redirect users to the event sign-in page for that specific event, where users will be able to get their attendance counted for a specific event. They will be prompted for different fields including their name and email address so that their attendance will be properly counted. Users will be able to return to the home page using the back arrow in their browser, or by clicking the home button on the top left of the page.

Details Page:

Item 2A: The club officer login button. Left clicking on this button will redirect the user to the sign in page for the club officer area where they will be asked for both a username and password. If a club member/ individual without login credentials mistakenly navigates here, they will be able to return to the home page with the back button in their web browser.

Item 2B: The home page button. Left clicking on this button will redirect users to the website home page.

Item 2C: Event Title. This text field contains the title of the event.

Item 2D: Event Location. This text field contains the location of the event.

Item 2E: Event Details. This text field contains information about the event.

Item 2F: Event Time and date. This text field contains the start time and date of the event and the event time and date of the event.

Item 2G: Attendees list. This will contain a list of all the people who have signed in to a particular event.

Item 2H: The back to list button. Left clicking on this button will redirect users to the website home page.

 

Event Check-In Page:

Item 3A: The club officer login button. Left clicking on this button will redirect the user to the sign in page for the club officer area where they will be asked for both a username and password. If a club member/ individual without login credentials mistakenly navigates here, they will be able to return to the home page with the back button in their web browser.

Item 3B: The home page button. Left clicking on this button will redirect users to the website home page.

Item 3C: User email address field. The user must enter a valid email address in order to sign in to an event. If a user enters an invalid email address, they will be given a warning that the email they entered is invalid, and will be unable to click the Sign In button.

Item 3D: Event Password Field. The user must enter the valid event password chosen by the club officers for the event. The officers should distribute the event password to attendees of the event in order to verify that they actually came to the event. If the user enters the incorrect event password, they will be given a warning that the event password they entered is incorrect, and they will be unable to click the Sign In button.

Item 3E: First name Field. Users must enter their first name for record of their attendance.

Item 3F: Last name Field. Users must enter their Last name for record of their attendance.

Item 3G: UIN field. Users must enter a valid UIN for record of their attendance. If the number entered is not a valid UIN (i.e. too short or too long to be a UIN) the user will receive a warning that they have not entered a valid UIN and will be unable to click the Sign In button.

Item 3H: Sign In button. Users should left click this button when they have correctly filled in all of the above fields for their attendance at an event to be recorded. If they click this button while one or more of the fields is incorrectly filled out, they will be informed and given the opportunity to correct that field before clicking this button again. After successfully signing in, users will be redirected to the home page.

Item 3J: The back to list button. Left clicking on this button will redirect users to the website home page.

Item 14L: Last Name Column. Displays the Last Name of the attendee.

Item 14N: Created At Column. Displays the time that the attendee signed in.

Item 14P: Updated At Column. Displays the last time that the information on this sign in was edited.

Item 14Q: Event ID column. Contains the event ID number that this participation was for. This number links the attendance record to a specific event. Users can check the events page at 14C to find the event ID of a specific event.

This CSV can then be opened in google sheets or Microsoft excel so that the club officers can hang on to attendance records from previous semesters.
